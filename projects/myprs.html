myPRs

The project

Completed February 2020. This application is a mobile friendly website that tracks fitness personal records. A user can create an account and log into the application to record fitness goals. myPRs has three categories of movements: Lifts, Cardio and Skills. The application is already loaded with predefined movements in each category and the user can create new movements to add to their list. Within each specific movement, the user can add new 1 rep max logs, edit the log or delete it. I was inspired to create this project because I needed a way to track my own fitness personal records with a specific, easy to use application. There are many other beautiful fitness application on the market but they tend to get very busy with too many options and I just wanted a simple quick way to look at my person fitness records.

Focus

My focus on this project was to build my first full stack application. I learned Node.js for the backend and MongoDB for my database. I used Balsamiq to wireframe my design and lucid charts to map out my data. Postman was used to test all of my endpoints throughout development. Also for the front end I didn’t not want to depend on any libraries so this application is built in Vanilla JavaScript (ES6+), HTML5 and CSS3.

What I learned

This is my first full stack multi-tenant application. I learned how to develop a RESTful backend in Node.js / Express, and MongoDB; while developing a mobile first experience in vanilla JavaScript. In MongoDB I set up a database with MongoDB Atlas, create schemas and validation. In Node.js I learned how to create RESTful APIs, add in user authentication and utilize other libraries through NPM. I leveraged the Heroku PaaS hosting platform.
