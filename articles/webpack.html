Getting Started with Webpack

July 15, 2019
Intro

Today we are going to talk briefly about what webpack is and specifically I am going to do a walk through on how to set up your first Webpack project. For this article you need to already know some basics like: HTML, writing simple JS functions that can print a string to the console, how to view your code in the browser and lastly you must already have Node Package Manager installed. If you need NPM please stop here and download/familiarize yourself with it before you can continue with this tutorial

Webpack is a module bundler that runs during the development process. This is useful because applications are much more complex than they used to be. We have more script and css files and more dependencies. When we use something like webpack, it bundles up our files and we only have to reference that bundled file instead of all of the separate files. You can use Webpack for your JavaScript and your CSS but today I am just going to talk about JS. We will walk through a step by step process on setting up the basics of Webpack that can get you started. to get you started on Webpack.

Prove Why Webpack is Cool

I want you to do a small walk through of how scoping and asynchronous loading works. Create an home.html file and add your normal boilerplate. Then create a src folder and in that add index.js file and test.js file. Add the reference to the js files listing ./src/index.js first in the HTML. In the test.js create a function that prints a string in the console. My function is called sayHello and will print a message in the console when it is called. Call this function in the index.js file

What you should see in the console is an error like this: Uncaught ReferenceError: sayHello is not defined This is to be expected because the order your list your script file matters. index.js is a dependency of test.js. You can’t call the function in the first file before it is declared it in the second file. Go ahead and switch the script tags in the HTML so test.js is referenced first. Reload the browser and your code should work now. Imagine your application has over 20 script files (which would be very common) it can be very challenging to keep it all straight, especially when you have dependencies in your logic. We are not going to spend much more time on this concept but the point being is that with a bundler like Webpack, you don't have to worry about this. Read on to see a step by step instructions on setting up Webpack.

Set Up Locally
  1. Initialize NPM. Open up your terminal. If you are using VSCode shortcut is ctrl+shift+`
    • Enter in the command line: npm init -y

    This pulls in the default package.json for our program. You will see this file added to the root level

  2. Install Webpack:
    • Enter in the command line: npm install webpack webpack-cli --save-dev

    This is saved as a development dependency.
    Added the node_modules folder and package-lock.json file

    ** Note: I am using a plugin from VSCode called ‘Simple Icons’ for my icons

  3. Create a webpack.config.js file in the root directory of project (meaning same level as your home.html). This is where we will specify our configurations. We need to require ‘path’ which is module core element from Node.js. You can read more about path here with their docs if you would like. Create a module.exports object. In this object we need to create an entry point, an output and a mode
  4. Go to package.json file and specify a script: “build”: “webpack”. Don’t forget to add the comma after the “test” value.
Put Webpack to Use
  1. In the command line enter: npm run build

    2. This will add the dist folder with bundle.js

  2. We now need to reference that new file. This bundle file is all of your code bundled into one file.

    3. - Remove the test.js and index.js in the HTML
    - Add /dist/bundle.js script reference to the HTML .

  3. We need to import and export our functions. If you are unfamiliar with Import/Export you can check out my article here.

    4. - Go to test.js and export your function
    - Go to index.js and import that function

  4. Each time you make changes to the code and want to test in the browser you need to run webpack again so it can re-bundle you files. In the command line enter: npm run build You should now see your function message in the console.

Last piece I want you to try on your own. Create another function in test.js that just console.logs a string. Make sure you export it then import it and call it in index.js. Now head to the command line and run the build command again. This should print both of your strings in the console. If you are having any trouble my full code is on GitHub

Conclusion

Today we did a short introduction to Webpack and why we would want to use it. We did a step by step setup of Webpack for a simple application that just prints messages to the console. We learned the three main commands that is needed for Webpack:

  1. npm init -y To initialize NPM and pulls in the default package.json
  2. npm install webpack webpack-cli --save-dev To install Webpack and creates node_modules folder and the package-lock.json file
  3. npm run build Which runs Webpack

We learned how to create a webpack.config.js with the requirements and exports that live in this file as well as what we need to add in package.json. Lastly we learned that we needed to use import and export in our JS files.

I hope you found this tutorial helpful and I encourage you to continue learning more on Webpack. We really just scratched the surface on Webpack but hopefully this got you started on a working project.
