ES6 Import and Export

Jan 23, 2018
Introduction

Today we are going to talk about JavaScript modules. This is a newer feature with ES6 and modules allow us the use of Import and Export in our JS files. If you want to keep you code base organized then read on because this is actually a pretty easy concept to get started with. I am going to go over why we would use modules with import and export, where we would use it. We will also review some sample code so you can understand the syntax and get started on your own project.

Why Modules?

Think about any textbook you have read. Is all of the context in one big chapter or is it divided up into many chapters? I am assuming you answered that the textbook was divided up into chapters; we can think of code modules in a similar way. It is a way to organize your code and allows you to share data between scopes. Think of the term modules more as a concept and the specific syntax is when we would use the keywords import and export. At this point you are probably familiar with scope in JavaScript but if you need a quick refresher check out the MDN docs. As your application grows, it can be harder to maintain when everything is in one giant file. Modularizing your code base is a clean way to group together related logic and can also be helpful when debugging. If you read my earlier article on JS Classes where I built a small game with a Monster Class, Hero Class and then the interacting logic, you might have noticed that the very small application grew pretty quickly. You can read that article here to take a quick glance at the code or just continue on with this article. I want to take that earlier example and break it up into 3 separate files: Monster.js, Hero.js, and main.js. This is where Import and Export will come into play. Its allows me to write my code in separate files but share what I need with other files.

Getting Started

This is the “easy” part. We will dive a little deeper into the syntax and many of the options a little later but your main setup is as follows:

  1. Set up an index.html file with your normal boilerplate
  2. Create a main.js file and include in your HTML before your closing body tag
  3. Create a Monster.js file
  4. Create a Hero.js file
  5. Notice I did not mention to add those last two js file in the index.html
  6. Inside main.js script tag add type=”module”
    • type=”module” is all you need for the browser to treat the external script (or inline script) as a ES6 module

Definitions

Module in JavaScript is more or less just a collection of new features that were added in ES6. Yes there are plenty of libraries and frameworks like Webpack and Babel that can do the same thing but modern browsers can now support modules which can optimize load time without the use of other libraries. A few things to keep in mind with modules:

  • Modules have a lexical top level scope so creating a variable with var does not create a global variable
  • import and export are only available within modules
  • You can export functions, variables and classes

Strict Mode

Imported modules are in strict mode whether you declare it or not. Before we jump into the code I wanted to review what strict mode is. “use strict”; helps developers to write cleaner code. Here are a few benefits of using strict mode:

  • It will throw errors that otherwise would be silent
  • It can fix mistakes that make it difficult for JS engines to perform optimizations
  • Can prohibit syntax that will likely be defined in future versions of ES6
  • Won’t allow you to use a variable without declaring it
  • Deleting a variable/object/function is not allowed
  • Duplicating a variable name is not allowed
  • Makes it easier to write secure JavaScript

Import

We just set up the export of what we wanted to share with the import file but before we can access that information we need to tell our file what is coming in. Its is kinda like the Monster.js file threw us some data and now in the import file we need to catch it. Import is a statement that will go on the top line of your code where we use the keyword import, tell the browser what it is we are importing and where it is from. Open back up the main.js file. We are going to import a variable, function and Monster class to the main.js file. The syntax breakdown is as follows: import { whatYouAreImporting } from ‘filepath.js’; The file path is called the “module specifier” or the “import specifier”. After you have added the import statement(s) add a console.log("testing") in to prove that you now have access to your exports.

If you want to import everything from Monster.js we can use the import all syntax. For example instead of specifically naming all of the imports like above, the syntax would be: import * as anyName from ‘./Monster.js’; The * means ‘all’ and anyName can be any name but it represents the name of the module object being imported. Conclusion

Today we talked about modules and the use of import and export. Using modules is a great way to keep you files clean and organized. I like to think of import and export as a way our files can communicate. We set up one JS file that will be reference in the HTML. We called this main.js and in the script tag we needed to add type=”module”. This is the file that you will be importing to. We learned that you can import data individually or all with the * syntax. Our Monster.js file is where we used the export syntax. We exported a variable, function and a class. Note that this file we did NOT add in the bottom of the HTML with a script tag. Try taking these steps and go a little bit further. Create a Hero.js file, add a Hero class and with a couple simple methods then import it to the main.js file just like we did with the Monster.js. Now in main.js you have access to your exports from Monster.js and Hero.js and you can have those two classes interacting in main.js.
